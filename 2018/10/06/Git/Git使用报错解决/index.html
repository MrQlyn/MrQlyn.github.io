Git使用报错解决

1、下载远程库时（git pull）报错：fatal: refusing to merge unrelated histories

原因：GitHub认为本地库和远端库是俩个不同的库，所以告知无法合并。

解决方法：

1、修改git pull语句使得俩个分支合并在一起

2、下载GitHub库，将改动文件添加到拉下来的远程库，然后再提交。这被GitHub看作是一次更新。

2、提交时（git add .）换行符在不同操作系统上出现转换问题：

解决方法：

具体查看这篇文章https://www.jianshu.com/p/450cd21b36a4

3、推送时（git push 远程库名 远程库分支名）出现错误：主要因为缺少readme.md不在代码目录中。

解决方法：

使用pull（fetch+merge）克隆远程项目库文件，也可以直接使用git clone下载文件

特殊情况：使用git pull –rebase origin master将本地代码覆盖。
